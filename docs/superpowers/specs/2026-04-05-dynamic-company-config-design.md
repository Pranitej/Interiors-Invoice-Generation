# Dynamic Company Config — Design Spec

**Date:** 2026-04-05  
**Author:** Vangala Pranitej  
**Status:** Approved

---

## Problem

All company identity data (name, address, phone, email, logo) for Skanda Industries is hardcoded across 8+ files. Adapting the project for a new client requires hunting down and editing every occurrence individually — error-prone and time-consuming.

## Goal

Introduce two config files (`client/src/config.js` and `server/config.js`) as the single source of truth for company identity. Changing one file should update the entire application — UI, invoices (Admin/Client/Temp/Compare), header, login page, PDF filenames, and server health check.

---

## Deployment Model

**One deployment per company.** To deploy for a new client:
1. Edit `client/src/config.js` with the new company's details.
2. Edit `server/config.js` with the server-side name.
3. Drop the company logo at `server/public/logo.png`.
4. Redeploy.

No multi-tenant support. No runtime switching.

## Config Files

### `client/src/config.js`

```js
const companyConfig = {
  name: "SKANDA INDUSTRIES",        // Display name (ALL CAPS for invoice header)
  tagline: "Your Dream, Our Design", // Shown on login page
  address: "123, Main Street, Bengaluru - 560001",
  phones: ["+91 98765 43210", "+91 91234 56789"], // array — supports multiple contact numbers
  email: "industry.skanda@gmail.com",
  website: "",                       // Optional — leave empty to hide
  logoFile: "logo.png",             // Filename served from server/public/
};

export default companyConfig;
```

### `server/config.js`

```js
const companyConfig = {
  name: "Skanda Industries",
};

export default companyConfig;
```

### Modified Files

#### `client/index.html`
- Remove hardcoded `<title>Skanda Industries</title>`
- Replace with a generic placeholder: `<title>Invoice System</title>`
- The actual title is set dynamically in `client/src/main.jsx` using `document.title = config.name`

#### `client/src/main.jsx`
- Import `companyConfig` from `./config`
- Set `document.title = companyConfig.name` on startup

#### `client/src/components/AdminInvoice.jsx`
- Import `companyConfig`
- Replace hardcoded logo URL: `${VITE_API_BASE}/public/${companyConfig.logoFile}`
- Replace `SKANDA INDUSTRIES` → `companyConfig.name`
- Replace hardcoded address → `companyConfig.address`
- Replace hardcoded phone → render `companyConfig.phones` joined as `" / "` (e.g. `+91 98765 43210 / +91 91234 56789`)
- Replace hardcoded email → `companyConfig.email`
- Replace footer `"For Skanda Industries"` → `"For ${companyConfig.name}"`
- Add `onError` logo fallback (see Logo Fallback section)

#### `client/src/components/ClientInvoice.jsx`
- Same substitutions as AdminInvoice (phones rendered as joined string)
- Replace thank-you line: `"Thank you for considering ${companyConfig.name}..."`
- Footer email → `companyConfig.email`

#### `client/src/components/ClientInvoiceTemp.jsx`
- Same substitutions as AdminInvoice (phones rendered as joined string)
- Replace thank-you line: `"Thank you for considering ${companyConfig.name}..."`
- Footer email → `companyConfig.email`

#### `client/src/components/CompareInvoices.jsx`
- Same logo, name, email substitutions
- Replace footer: `"...generated by ${companyConfig.name} Invoice System"`

#### `client/src/components/Header.jsx`
- Import `companyConfig`
- Replace hardcoded `"Skanda Industries"` → `companyConfig.name`

#### `client/src/pages/Login.jsx`
- Import `companyConfig`
- Replace hardcoded `"Skanda Industries"` → `companyConfig.name`
- Show `companyConfig.tagline` below the company name

#### `client/src/pages/History.jsx`
- Import `companyConfig`
- Replace PDF download filename: `` `${companyConfig.name}-${type}-Invoice-${id.slice(-6)}.pdf` ``

#### `server/server.js`
- Import `companyConfig` from `./config.js`
- Replace health check response: `{ ok: true, message: \`${companyConfig.name} API\` }`

## Logo Fallback

When `server/public/logo.png` does not exist or fails to load, every `<img>` for the logo uses an `onError` handler to swap to a placeholder:

```jsx
const LOGO_PLACEHOLDER = `data:image/svg+xml;utf8,<svg xmlns='http://www.w3.org/2000/svg' width='120' height='60'><rect width='120' height='60' rx='6' fill='%23e2e8f0'/><text x='50%25' y='55%25' dominant-baseline='middle' text-anchor='middle' font-family='Arial' font-size='11' fill='%2364748b'>No Logo</text></svg>`;

<img
  src={logoURL}
  onError={(e) => { e.target.onerror = null; e.target.src = LOGO_PLACEHOLDER; }}
  alt={companyConfig.name}
/>
```

This placeholder is an inline SVG — no network request, no broken image icon.

## What Does NOT Change

- `client/.env` and `server/.env` — infrastructure config (API base URL, MongoDB URI), not company identity
- `client/src/json/roomConfig.json` — no company data
- All pricing, calculation, and invoice data logic — untouched
- Database seeding (`server/database/seeding.js`) — user accounts are not company-specific
- Auth flow — unchanged
